- **达标线**：50 次零失败，或失败率 < 2%

### 2. 自动修订震荡问题
- **现状**：头条 2837 字 → 修订到 740 字（过头） → 第二轮扩充到 1352 字。修订一轮砍太多说明修订指令不够精确
- **目标**：修订后字数在目标范围的 ±20% 内，不要出现"砍过头再补回来"的震荡
- **方法**：
  - 修订指令里给一个具体的"目标字数"而不是"精简到 X 字左右"
  - 加一个保护逻辑：如果修订后字数 < 目标下限的 80%，自动触发扩充轮
  - 或者把 max_rounds 从 2 提到 3，给更多修正空间

### 3. 结尾重复问题根治
- **现状**：prompt 里加了"不要多个最后一句"的禁令，但 LLM 不一定遵守
- **目标**：结尾不超过 150 字，只有一个收尾句 + 一个 CTA
- **方法**：
  - 在 `_generate_ending` 的 `max_tokens` 设为 512（硬限制输出长度）
  - 或者把结尾单独拆成两步：先生成金句收尾（1-2 句），再生成 CTA（1-2 句），分别控制字数
  - 在自动修订里加一条检查：如果结尾超过 200 字，自动触发精简
